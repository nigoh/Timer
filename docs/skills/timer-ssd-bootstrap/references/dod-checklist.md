# Definition of Done (DoD)

Timerリポジトリで要件整理・仕様分解・実装計画を完了とみなすためのチェックリスト。

## 1. ドキュメント更新
- [ ] 変更内容が仕様変更を伴うか判定した。
- [ ] 仕様変更ありの場合、`docs/REQUIREMENTS.md` を更新した。
- [ ] 仕様変更ありの場合、関連仕様書（`docs/TECHNICAL_SPECS.md` / `docs/TECHNICAL_SPECS_SHADCN.md` など）を更新した。
- [ ] 仕様変更なしの場合、PR本文に「仕様影響なし」の根拠を記載した。
- [ ] `references/requirements-map.md` の対応表を最新化した。

## 2. テスト観点整理
- [ ] 正常系（開始/停止/リセット等）の検証観点を定義した。
- [ ] 異常系（不正入力、状態不整合、復元失敗等）の検証観点を定義した。
- [ ] 境界値（0秒、長時間、複数同時実行）の検証観点を定義した。
- [ ] 既存テストの再実行対象（`src/features/timer/stores/__tests__`）を明示した。

## 3. 影響範囲確認
- [ ] 変更対象の `src/features/timer/**` を列挙した。
- [ ] UI・状態管理・永続化への影響有無を切り分けた。
- [ ] 依存ドキュメント（要件/仕様/機能一覧）への波及を確認した。
- [ ] リリースノートや運用手順への反映要否を判断した。

## 4. 実装計画の品質
- [ ] タスクが「1コミットでレビュー可能」な粒度に分割されている。
- [ ] 先行タスク（仕様更新）と後続タスク（実装・検証）の順序が定義されている。
- [ ] 完了条件（完了判定コマンド、期待結果）が各タスクに記載されている。
