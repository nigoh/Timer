# NON_FUNCTIONAL_VERIFICATION

## 目的
変更後の品質を、軽量かつ再現可能な手順で確認する。

## 最低確認
1. `npm run type-check`
2. `npm run test`
3. `npm run build`

## 重点観点
- 型安全性: TypeScript エラー 0 件
- 安定性: 主要タイマー操作でランタイムエラーが出ない
- ログ: エラー発生時に LogViewer で確認できる
- 通知: 権限なし環境でもタイマー動作が継続する

## 変更種別別チェック
### ストア変更
- tick 処理が多重実行されないこと
- 既存セッションデータが破壊されないこと

### UI変更
- 4タブすべてで主要操作が到達可能なこと
- モバイル幅でボタンが操作不能にならないこと
- GitHub Issue 起点の会議下書き生成で、候補編集前に確定保存されないこと

### ユーティリティ変更
- `notification-manager` の初期化失敗でアプリが停止しないこと
- `logger` の保存失敗でアプリが停止しないこと

### 外部入力取り込み変更
- GitHub API 失敗時でも会議の手動作成が継続可能であること
- Issue 本文の解析結果が空でも会議名候補のみで作成フローを続行できること

### AI API 設定・連携変更
- API キーが永続化されないこと（ページリロードで再入力が必要なこと）
- provider/model/apiKey 未設定時に設定エラーが検出されること
- AI API 呼び出しが失敗しても既存のルールベース提案が利用可能であること
