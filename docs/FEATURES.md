# FEATURES

## 画面構成

- タブ: 基本タイマー / ポモドーロ / アジェンダ / 複数タイマー
- 共通: ログビューア、エラーバウンダリ、フッター

## 機能一覧

### 基本タイマー

- 時間設定、開始/一時停止/停止/リセット
- セッション履歴管理（削除/全削除）
- 履歴エントリごとの GitHub Issue 紐付け（追加/解除）
- Issue タイトル自動取得（Owner/Repo + Issue番号入力時、GitHub API 利用）
- GitHub Issue 別の合計作業時間/件数の集計表示

### ポモドーロ

- 作業/短休憩/長休憩の状態遷移
- サイクル管理、統計表示
- 自動開始設定

### アジェンダタイマー

- 会議・議題の CRUD
- 議題の並べ替え、進行、超過判定
- 会議設定（通知・ベル・サイレント）
- 会議ごとの GitHub Issue 紐付け（追加/解除、Issue タイトル自動取得）
- GitHub Issue を起点にした会議下書き生成（会議作成ダイアログで会議名・アジェンダ候補を自動入力）
- 下書きアジェンダの取り込み対象をチェック選択して会議作成
- GitHub Issue 入力用テンプレート（`docs/templates/GITHUB_MEETING_INPUT_ISSUE_TEMPLATE.md`）に準拠した取り込み

### 複数タイマー

- 複数タイマーの同時管理
- カテゴリ管理
- 個別操作・一括操作

### 会議レポート

- 会議データからレポート草稿を生成
- AIアシスト提案（議事録要約 / 合意形成 / ファシリテーション / 次回アジェンダ / 会議事前準備）を表示し、下書きへ反映
- AI API 設定の段階導入（provider/model/api key/temperature/接続テスト、初期はUI未接続スケルトン）
- ToDo 管理
- Markdown 出力
- 会議に紐付いた GitHub Issue へのコメント投稿（会議レポート確認ダイアログから手動投稿）
- 差分のみ投稿モード（前回レポートとの差分行のみ Issue コメント投稿）
- Issue チェックリストから ToDo（担当/期限）を自動反映
- 投稿前プレビューと投稿テンプレート選択（要約/詳細）
- 投稿成功した Issue コメント URL の履歴保存

### ログ管理

- レベル別ログ記録（ERROR/WARN/INFO/DEBUG/TRACE）
- LocalStorage 保持
- AI 解析用プロンプト生成

### 分析ダッシュボード

- 日次/週次/月次の集中時間・完了セッション数の可視化
- ポモドーロ達成率、休憩比率、連続達成日数（ストリーク）の表示
- 会議の時間超過率、議題ごとの予定対実績の差分分析
- 時間帯・曜日ごとの生産性ヒートマップ表示
- 目標値（例: 1日120分集中）に対する進捗トラッキング
- フィルタ（期間、タイマー種別、カテゴリ、会議）と比較表示
- CSV/Markdown 形式でのサマリーエクスポート
