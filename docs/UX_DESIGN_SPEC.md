# UX_DESIGN_SPEC

## 画面レイアウト


- 左サイドバー（shadcn Sidebar、`collapsible=icon` モード、開閉トグル付き）: タスク一覧（ドラッグ並べ替え対応）、タスク追加、設定・ログ画面へのリンク、ダークモード切り替え。デスクトップではパンくずナビ（Breadcrumb）でタスク名/設定画面を表示。モバイルではヘッダーバーで画面名を表示。コマンドパレット (Ctrl+K / Cmd+K) でタスク検索・切り替え・アクション実行が可能。
  - モバイル幅では常時表示せず、ヘッダーの「メニュー」ボタンから機能選択を行う
  - タイマー: 基本タイマー / ポモドーロ / 複数タイマー のサブタブ
  - 分析: ダッシュボード（KPI / トレンドチャート / ヒートマップ / ドーナツチャート）を独立メニューとして表示
- 中央メイン: 各タイマー機能のメインビュー
- 下部: フッター


## UX 原則

- 主要操作（開始・停止・リセット）は 1 アクションで実行可能にする。
- 時間経過・完了・超過は視覚的に認識しやすくする。
- エラー時はアプリ全体を停止させず、ErrorBoundary で隔離する。


## インタラクション

- タブ切り替え時はコンテキストを維持しつつ、アクティブ機能を明示する。
- 通知はユーザー設定（サイレント/音）を尊重する。
- 長文入力（議事録・メモ）は途中編集可能な UI を維持する。
- 会議作成では「GitHub Issue を入力して下書き生成」→「候補確認・編集」→「確定作成」の 3 ステップで誤投入を防ぐ。
- Issue 取得に失敗した場合は、原因を短文で提示しつつ手動入力フォームへ遷移せず同画面で継続できる。
- AI API 設定は「入力→接続テスト→保存」の順で完了できる導線とし、保存前に疎通可否を明示する。

- 会議AIアシスト利用時は、提案内容を自動確定せず「確認して反映」操作を必須にする。

## レスポンシブ

- モバイル幅でも主要ボタンと進行情報を優先表示する。
- 補助情報は必要に応じて折りたたみ/省略表示する。
- 会議画面のウィジェットはモバイル幅で1カラム表示にして、操作ターゲットを確保する。

## クラウド認証・同期 UX

### ログインモーダル

- サイドバーフッターに「ログイン」ボタン（枠付き、ログインアイコン付き）を表示する。
- ボタンタップでモーダルが開き、GitHub・Google の 2 択を縦並びで表示する。
- Supabase 未設定（環境変数未設定）の場合はボタン自体を非表示にする。
- ログインはリダイレクトフロー（ポップアップなし）とする。

### ユーザーメニュー（ログイン後）

- サイドバーフッターにアバター画像・表示名を横並びで表示（shadcn Avatar + Button）。
- クリックでドロップダウンを開き、以下を表示する:
  - ユーザー名 + メールアドレス（ラベル）
  - 連携プロバイダー（GitHub / Google）表示（非操作）
  - 「ログアウト」（赤テキスト、ログアウトアイコン付き）
- サイドバーが折りたたみ（icon モード）の場合はアバターのみ表示する。

### 同期ステータスインジケーター

ログイン中のみサイドバーフッターに小さいアイコンで表示する。

| ステータス | アイコン | 色クラス |
|-----------|----------|---------|
| idle（同期済み） | Cloud | text-muted-foreground |
| syncing（同期中） | Loader2（スピン） | text-info |
| error（エラー） | CloudAlert | text-destructive |
| offline（オフライン） | CloudOff | text-warning |

- ホバーでツールチップ: ステータスラベル + 最終同期時刻（HH:MM:SS）を表示する。
- サイドバー展開時はアイコン横にラベルテキストも表示する。

### ゲストデータ移行通知

- 初回ログイン後にゲストデータの移行処理を実行し、完了時に通知する。
  - 通知文: 「データ同期完了 — N 件のデータをクラウドに移行しました」
  - 移行データが 0 件の場合は通知しない。
