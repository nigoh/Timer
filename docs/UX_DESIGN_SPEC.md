# UX_DESIGN_SPEC

## 画面レイアウト
- 上部: タブナビゲーションとログボタン
- 中央: 各タイマー機能のメインビュー
- 下部: フッター

## UX 原則
- 主要操作（開始・停止・リセット）は 1 アクションで実行可能にする。
- 時間経過・完了・超過は視覚的に認識しやすくする。
- エラー時はアプリ全体を停止させず、ErrorBoundary で隔離する。

## インタラクション
- タブ切り替え時はコンテキストを維持しつつ、アクティブ機能を明示する。
- 通知はユーザー設定（サイレント/音）を尊重する。
- 長文入力（議事録・メモ）は途中編集可能な UI を維持する。

## レスポンシブ
- モバイル幅でも主要ボタンと進行情報を優先表示する。
- 補助情報は必要に応じて折りたたみ/省略表示する。


## 会議前ファシリテーション UX
- 会議作成後、開始前に「目的種別」「期待アウトプット」「合意条件」を1画面で入力できる。
- 参加者の事前コメントは、提案/懸念/確認事項のカテゴリで収集し、論点一覧に自動整形する。
- 「会議前ブリーフを生成」操作で、開始時に司会者が確認すべきチェック項目を表示する。
- 議題終了時に未解決項目を Parking Lot へ移し、会議終了時に次回議題へ転記できる導線を提供する。
