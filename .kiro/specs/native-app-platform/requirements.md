# 要件定義: モバイルアプリ・Windows アプリケーション化

## はじめに

Focuso は現在 React + Vite によるウェブアプリケーションとして提供されている。
本機能では、同一コードベースからモバイルアプリ（Android / iOS）および Windows デスクトップアプリケーションを生成する仕組みを導入し、
ネイティブ配布・オフライン動作・OS 通知統合を実現する。

## 背景と動機

- ウェブ版ではブラウザタブを常時開いている必要があり、バックグラウンド動作やOS通知に制約がある
- Windows ユーザーはデスクトップアプリとして常駐させたいニーズがある
- モバイルユーザーはホーム画面から直接起動し、外出先でもタイマーを利用したい
- 既存のReactコードベースを最大限再利用し、開発コストを抑えたい

## 要件

### 要件 1: Windows デスクトップアプリケーション

**目的:** ユーザーとして、Focuso を Windows デスクトップアプリケーションとして利用したい。そうすれば、ブラウザを意識せずに常駐起動でき、OS ネイティブの通知と統合できる。

#### 受け入れ基準

1. When ユーザーがインストーラーを実行する, the アプリ shall Windows 上にインストールされ、スタートメニューとデスクトップにショートカットが作成される
2. When アプリが起動する, the システム shall 既存のウェブ UI をそのまま表示し、全機能が動作する
3. When タイマーが完了する, the システム shall Windows のネイティブ通知（トースト通知）を送信する
4. When ユーザーがウィンドウを閉じる, the アプリ shall システムトレイに最小化し、バックグラウンドでタイマーを継続できる
5. When オフライン環境である, the アプリ shall ネットワーク接続なしで基本機能（タイマー・ポモドーロ・履歴）が動作する

### 要件 2: モバイルアプリケーション（Android / iOS）

**目的:** ユーザーとして、Focuso をスマートフォンのネイティブアプリとして利用したい。そうすれば、外出先でもタイマーを手軽に使え、プッシュ通知で完了を知ることができる。

#### 受け入れ基準

1. When ユーザーがアプリストアからインストールする, the アプリ shall ホーム画面にアイコンが追加される
2. When アプリを起動する, the システム shall モバイル画面に最適化されたレスポンシブ UI を表示する
3. When タイマーが完了する, the システム shall OS のプッシュ通知を送信する（アプリがバックグラウンドでも動作）
4. When デバイスがオフラインである, the アプリ shall 基本機能がローカルで動作する
5. When ユーザーがアプリをバックグラウンドにする, the タイマー shall 正確に動作を継続する

### 要件 3: コードベース共有

**目的:** 開発者として、ウェブ・デスクトップ・モバイルで同一の React コードベースを共有したい。そうすれば、機能追加やバグ修正を一度の変更で全プラットフォームに反映できる。

#### 受け入れ基準

1. Where ビルドシステムが構成される, the システム shall 単一のソースコードからウェブ・デスクトップ・モバイルの各ビルドを生成できる
2. Where プラットフォーム固有の機能がある, the システム shall プラットフォーム検出によりネイティブ API とウェブ API を切り替える
3. Where CI/CD パイプラインが構成される, the システム shall 各プラットフォーム向けのビルド・配布を自動化する

### 要件 4: データ永続化と同期

**目的:** ユーザーとして、どのプラットフォームでも同じデータ（設定・履歴）にアクセスしたい。そうすれば、環境を切り替えても作業の継続性が保たれる。

#### 受け入れ基準

1. Where デスクトップアプリである, the データ shall ローカルファイルシステムに永続化される
2. Where モバイルアプリである, the データ shall デバイスのローカルストレージに永続化される
3. When 将来的にクラウド同期が導入される, the システム shall ローカルデータとリモートデータのマージ戦略を持つ（将来拡張ポイント）

### 要件 5: 自動アップデート

**目的:** ユーザーとして、アプリが自動的に最新版に更新されてほしい。そうすれば、手動でアップデートする手間が省ける。

#### 受け入れ基準

1. When 新しいバージョンが利用可能になる, the デスクトップアプリ shall ユーザーに更新通知を表示する
2. When ユーザーが更新を承認する, the アプリ shall バックグラウンドでダウンロードしインストールを実行する
3. Where モバイルアプリである, the 更新 shall 各アプリストアの標準メカニズムに従う
