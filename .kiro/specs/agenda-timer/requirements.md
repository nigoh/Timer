# 要件定義: アジェンダタイマー

## はじめに

アジェンダタイマーは、会議の議題進行を時間管理する機能である。
議題ごとの予定時間設定、進行状況のリアルタイム表示、通知制御、議事録入力、
音声文字起こし、GitHub Issue 連携による会議下書き生成を提供する。

## 要件

### 要件 1: 会議・議題管理

**目的:** 会議主催者として、会議と議題を作成し議題ごとに予定時間を設定したい。そうすれば、計画的な会議進行ができる。

#### 受け入れ基準

1. When ユーザーが新しい会議を作成する, the システム shall 会議名と議題リストを入力できるフォームを表示する
2. When ユーザーが議題を追加する, the システム shall 議題タイトルと予定時間（分単位）を設定できる
3. The システム shall 議題の追加・編集・削除・並べ替えを提供する
4. The システム shall 会議全体の合計予定時間を自動計算して表示する

### 要件 2: 議題進行表示

**目的:** 会議参加者として、現在の議題の進行状況をリアルタイムで確認したい。そうすれば、時間配分を意識した議論ができる。

#### 受け入れ基準

1. While 会議が進行中である, the システム shall 現在議題の残り時間をリアルタイムで表示する
2. While 議題が予定時間を超過している, the システム shall 超過状態を視覚的に強調表示する
3. When 議題の予定終了 5 分前になる, the システム shall 5 分前通知を送信する（設定が有効な場合）
4. When 議題の予定時間が終了する, the システム shall 終了通知を送信する（設定が有効な場合）
5. When 議題が超過状態に入る, the システム shall 超過通知を送信する（設定が有効な場合）

### 要件 3: 会議設定

**目的:** 会議主催者として、自動遷移やサイレントモードなどの会議設定を制御したい。そうすれば、会議の種類に応じた最適な進行ができる。

#### 受け入れ基準

1. Where 自動遷移設定が有効である, the システム shall 議題完了時に次の議題へ自動遷移する
2. Where サイレントモードが有効である, the システム shall 通知音を抑制する
3. The システム shall ベル種別（なし / 1 回 / ループ）を選択できる
4. The システム shall 会議設定を永続化する

### 要件 4: 議事録入力

**目的:** 会議参加者として、議題ごとに議事録を入力したい。そうすれば、会議内容を構造的に記録できる。

#### 受け入れ基準

1. Where 議事録入力機能が含まれる, the システム shall Quill リッチテキストエディタを提供する
2. While PC 幅で表示されている, the システム shall フルツールバー付きのエディタを表示する
3. While モバイル幅で表示されている, the システム shall 操作性に応じた簡易ツールバーを表示する
4. The システム shall 各議題の議事録を個別に保持する

### 要件 5: 音声文字起こし

**目的:** 会議参加者として、発言をリアルタイムで文字起こしし議事録に挿入したい。そうすれば、議事録作成の手間を削減できる。

#### 受け入れ基準

1. When ユーザーが文字起こしボタンを押す, the システム shall Web Speech API を使用して音声認識を開始する
2. While 音声認識が実行中である, the システム shall 認識結果をリアルタイムでプレビュー表示する
3. When ユーザーが認識結果を確認して挿入ボタンを押す, the システム shall 結果を現在議題の議事録フィールドに挿入する
4. If 認識結果をユーザーが確認していない, then the システム shall 自動反映を行わない
5. The システム shall 日本語（ja-JP）と英語（en-US）の言語切り替えを提供する
6. If ブラウザが Web Speech API に非対応である, then the システム shall 文字起こしボタンをグレーアウト表示し、手動入力フローを継続可能にする

### 要件 6: GitHub Issue 連携（会議下書き生成）

**目的:** 会議主催者として、GitHub Issue から会議名とアジェンダ候補を自動生成したい。そうすれば、会議準備の手間を大幅に削減できる。

#### 受け入れ基準

1. When ユーザーが `owner/repo` と Issue 番号を入力する, the システム shall GitHub API から Issue データを取得する
2. When Issue データが取得できる, the システム shall Issue タイトルを会議名候補として設定する
3. When Issue 本文にタスクリスト・見出し・箇条書きがある, the システム shall それらをアジェンダ候補として抽出する
4. When Issue 本文に `Duration: 10m` や `所要: 10分` の記載がある, the システム shall 予定時間候補として設定する
5. When 候補が生成される, the システム shall ユーザーが会議名・各アジェンダを編集確定できる画面を表示する
6. If Issue の取得に失敗する, then the システム shall エラーを表示し、既存の手動入力フローへフォールバックする
