# 要件定義: 会議レポート

## はじめに

会議レポートは、アジェンダタイマーの会議結果からドラフトレポートを自動生成し、
AI アシストによるファシリテーション支援を提供する機能である。
GitHub Issue へのレポート投稿も可能とする。

## 要件

### 要件 1: ドラフトレポート生成

**目的:** 会議主催者として、会議結果からドラフトレポートを自動生成したい。そうすれば、レポート作成の手間を大幅に削減できる。

#### 受け入れ基準

1. When 会議が完了する, the システム shall 会議結果からドラフトレポートを自動生成する
2. Where ドラフトレポートが生成される, the システム shall サマリー、決定事項、ToDo、次回アクションのセクションを含める
3. The システム shall 各セクションをユーザーが編集・加筆できる
4. When ユーザーが保存ボタンを押す, the システム shall レポートを Markdown 形式で保存する

### 要件 2: AI アシスト提案

**目的:** 会議参加者として、AI によるファシリテーション支援を受けたい。そうすれば、会議の質と効率を向上できる。

#### 受け入れ基準

1. Where AI API が設定済みである, the システム shall 議事録要約の AI アシスト提案を表示する
2. Where AI API が設定済みである, the システム shall 合意形成の AI アシスト提案を表示する
3. Where AI API が設定済みである, the システム shall 進行改善の AI アシスト提案を表示する
4. Where AI API が設定済みである, the システム shall 次回アジェンダの AI アシスト提案を表示する
5. Where AI API が設定済みである, the システム shall 事前準備の AI アシスト提案を表示する
6. The システム shall AI 提案をユーザーが編集して採用可否を判断できる
7. If AI API 呼び出しが失敗する, then the システム shall 既存のルールベース提案へフォールバックする

### 要件 3: AI API 設定

**目的:** ユーザーとして、AI API の接続設定を管理したい。そうすれば、AI アシスト機能を段階的に導入できる。

#### 受け入れ基準

1. The システム shall 設定画面から AI プロバイダー（OpenAI / Anthropic）を選択できる
2. The システム shall AI モデル名を設定できる
3. The システム shall API キーを入力できる（メモリ保持のみ、永続化禁止）
4. The システム shall temperature パラメータを設定できる
5. When ユーザーが接続テストボタンを押す, the システム shall API 接続の成否を確認して結果を表示する

### 要件 4: GitHub Issue コメント投稿

**目的:** 会議主催者として、会議レポートを GitHub Issue コメントとして投稿したい。そうすれば、会議結果をプロジェクト管理と連携できる。

#### 受け入れ基準

1. When ユーザーが投稿ボタンを押す, the システム shall 会議に紐付けた Issue へレポートをコメント投稿する
2. The システム shall 投稿はユーザーの明示操作時のみ実行し、自動投稿は行わない
3. When 投稿が成功する, the システム shall 成功通知をユーザーに表示する
4. If 投稿が失敗する, then the システム shall 失敗通知をユーザーに表示する
5. The システム shall 認証情報（PAT）を永続化しない
